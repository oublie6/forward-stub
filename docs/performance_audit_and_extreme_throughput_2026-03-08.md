## 代码审查与优化结果

### 已完成优化项

#### 1) pipeline 编译阶段预分配 stage 切片（减少构建期扩容开销）
- 文件：`src/runtime/compiler.go`
- 改造：`CompilePipelines` 中 `pipeline.Pipeline` 初始化时为 `Stages` 按 `len(stagesCfg)` 预分配容量。
- 价值：减少 `append` 引发的切片扩容与复制开销，尤其在 pipeline/stage 数量较多场景下能降低热更新构建成本。

#### 2) dispatch 多订阅路径去除临时 clones 切片（减少一次中间分配）
- 文件：`src/runtime/update_cache.go`
- 改造：多 task fan-out 时，不再先构造 `[]*packet.Packet` 临时切片；改为对非首 task 逐个 `Clone` 并直接投递，最后再投递原始包给首 task。
- 价值：保持功能语义与包生命周期安全前提下，减少额外中间对象分配。

### 已识别并规避的 bug 风险

- 风险：若“先投递原始包再 Clone”，原始包可能被首 task 提前释放，导致后续 Clone 使用已释放 payload。
- 处理：最终实现采用“先克隆后投递原始包”顺序，已通过回归测试验证。
